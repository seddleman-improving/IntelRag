# Localhost Ollama Configuration

This document describes what is needed to run the Consulting Intelligence Platform using the local Windows machine's Ollama installation instead of the Mac Mini.

---

## Current Local Ollama State (as of July 2026)

| Item | Value |
|---|---|
| Ollama version | 0.31.2 |
| Installed models | `llama3.2:latest` (3.2B, ~2 GB) |
| Missing models | `nomic-embed-text` (required for embeddings) |
| Status | Running at `http://localhost:11434` |

---

## Steps to Switch to Localhost

### 1. Pull the missing embedding model

```powershell
ollama pull nomic-embed-text
```

### 2. Update `backend/.env`

Change these three values:

```env
LLM_BASE_URL=http://localhost:11434
LLM_MODEL=llama3.2
EMBED_MODEL=nomic-embed-text
CHROMA_PERSIST_DIR=./chroma_db
```

### 3. Rebuild the vector index

Embeddings generated by one model are not compatible with another. After changing `EMBED_MODEL`, the Chroma index must be rebuilt:

```bash
# From /backend
uv run python scripts/ingest.py
```

### 4. Restart the backend

```bash
# Stop with Ctrl+C, then:
uv run uvicorn app.main:app --reload
```

### 5. Verify

```powershell
curl.exe -s http://localhost:8000/api/v1/health
```

Expected: `{"backend":"ok","llm":"ok"}`

---

## Expectations vs Mac Mini

| | Local Windows Machine | Mac Mini |
|---|---|---|
| Model | `llama3.2` (3.2B) | `qwen2.5:32b` (32B) |
| Answer quality | Lower — smaller model | Higher — larger model |
| Ollama version | 0.31.2 (older) | Current |
| Speed | Faster (smaller model) | Slower (larger model) |
| Compatibility | Should work with LangChain `OllamaLLM` | Confirmed working |

> **Note:** No code changes are required to switch between backends — only the `.env` file needs to be updated.

---

## Switching Back to Mac Mini

Restore `backend/.env` to:

```env
LLM_BASE_URL=http://192.168.100.224:11434
LLM_MODEL=qwen2.5:32b
EMBED_MODEL=nomic-embed-text
CHROMA_PERSIST_DIR=./chroma_db
```

Then rebuild the vector index again (same `ingest.py` command) and restart the backend.
